Introduction

In this lesson, you will learn how to arrange page elements using floats and positioning to create structured "layouts". Layout refers to the arrangement of elements on a page. This arrangement can be simple or complex, it all depends on the design and the number of elements being arranged. The strategy of floating elements to create layouts has been around for a long time, so many websites and webapps employ the techniques you'll find throughout this lesson.

Floating Elements  

If you become a front-end engineer, you will frequently be asked to create layouts using HTML and CSS. You'll be asked to arrange content on the page into columns and rows to form a pleasing design. To do this, you'll need a series of strategies and tools to create those arrangements in an efficient and stable way.

At this time, CSS provides several methods for creating layouts. Some of those methods have been around since the advent of CSS (floating) and some are more recent (flexbox).

For now, we'll focus on one of the "old school" methods for creating layouts. We'll use floats.

Once you become the amazing programmer you are destined to be, you may never again create your own layouts using floats. Millions of websites and web apps use this set of techniques and you'll need to be familiar with its quirks.

This lesson will discuss several topics, including:

  • The float property and other CSS properties related to floating elements.
  • The use of the float property to create simple layouts with columns and other modular elements.
  • The pitfalls of using floats to create layouts and how to overcome those pitfalls.

Floats  

To understand how to create columns using floats, we first need to understand the concept of "floating" elements and how to control the behaviors of elements that have been floated.

In a nutshell, floating elements means that other elements can "flow around" the floated element. It can be a bit confusing but let's look at a couple examples to clear things up.

The first example is a simple page with red box (pretend it's an image) and some text. Nothing in this code is floated. We are looking at the default behavior without floating.

Floats are easy to use. The float property has a few main values that determine the behavior of a floated element:

  • float: left; - The floated element will move as far to the left as possible and elements that come after the floated element will flow up and around it on the right.

  • float: right; - The floated element will move as far to the right as possible and elements that come after the floated element will flow up and around it on the left.

  • float: none; - Remove the float behavior from an element and restore its default behavior. Basically, if you float an element, you can "unfloat" it by setting its float property to "none".

Play with the provided code samples. Change the margin values and the float values to see what happens. Experimentation can help you understand what is occurring.

Clear has a few values that can help control which floated elements another element will respect:

  • clear: left; - Tells an element to clear any preceding elements that have been floated left.

  • clear: right; - Tells an element to clear any preceding elements that have been floated right.

  • clear: both; - Tells an element to clear any preceding elements that have been floated left or right.

The Problem with Floats  

Before we finally create a floated layout of any complexity, we need to address a problem with floats and how to fix it.

Collapsing Parents  

We've established that when an element is floated, it allows elements that come after it to "flow around it". The way that it does this is by "collapsing" it's height. This collapsing behavior is what allows other elements to flow up and around it.

Here is the problem: The effect of this is that the floated element is basically ignored by the parent when the parent is determining its own height.

To demonstrate this behavior, let's look at a couple examples. In both examples, we'll have two div elements. The red div will have an id of #inner and is the child of the other div #outer. #inner has a background color of red, and #outer has a background color of light gray. #outer also has some padding and a drop shadow, so you can see it more easily.

In the first example, "#inner" is not floated.

The Clearfix Hack  

So, the question should be: "How do we use floats without experiencing the collapsing parent problem?"

Luckily, we have several solutions to this layout problem. We'll talk about two: overflow:hidden and the .clearfix hack.

Overflow Hidden  

This is a "quick and dirty" method to force parent elements to expand downward and accommodate floated children. Sadly, it also has the effect of hiding child elements that have been positioned outside of its bounds, making it only appropriate when no child elements overflow the parent.

You can certainly use this method, but be aware that it has side effects.

The following example uses the same code as our above examples. The only change is that we've added overflow: hidden; to #outer.

The Clearfix Hack  

A more stable (and more universally applicable) method is called The Clearfix Hack. It involves the use of a generic class that can be applied to the parent of floated elements. It doesn't create the content clipping issues of overflow: hidden; and is more widely used.

You'll likely see it at every web job you take.

This is the CSS necessary to create the clearfix hack. It involves a single class (.clearfix) that generates a hidden element. That hidden element is placed after any of the parent's children. Then, the hidden element is used to clear all the floated children.

/*
This snippet of code should be added to your CSS
files so you can use the clearfix hack. Once this is in your CSS,
you can add the class to any elements that have floated children.
*/
.clearfix:after {
  content: "";
  display: block;
  clear: both;
}

If it's confusing, don't worry. It's confusing to lots of folks who have been programmers for some time. The behavior isn't intuitive. Take your time and review the following example, which uses the clearfix Hack to clear .inner.

Conclusion  

Float-based layouts are falling out of fashion. In time, this method of laying out pages will likely go the way of the dinosaur. Floating and clearing are essential page layout skills that you will need as a front-end developer.

Determining Stacking Order with z-index  

Sometimes, elements overlap. Its a natural consequence of positioning and layout. If you move things around the page, they might overlap with other elements. Often, we want to control how things overlap and determine which elements come to the front and which fall to the back. The order in which elements render on top of one another is called "stacking order". CSS provide the z-index property to control stacking order.

In this article, we will discuss the z-index property, how to apply it to elements, and when it is effective.

Visualizing z-index  

Before we start digging into code, let's try to conceptually understand z-index by exploring the analogy of a stack of papers.

Imagine a stack of four sheets of papers on a desk. You are looking downward on the pages and can see that they overlap one another. One sheet of paper is on the bottom, and one is on top. The other two pages are somewhere in between. This illustrations of concept of stacking order. You can clearly see that the papers have an order from top to bottom.

Now, imagine that we assign each sheet an integer indicating where it appears in the stack. To keep the ordering simple, we'll give the bottom sheet of paper the lowest number 0. As we move up the stack, we'll give each paper larger numbers until we reach the top sheet of paper. The top sheet of paper will be assigned the largest number. Finally, we'll call these integers "z-index".

The specific numbers don't matter. We could have assigned the bottom page the number 10 or -1 or 100. So long as the next page up the stack has a larger number it will appear on top. A element with a z-index of 101 will appear above of an element with a z-index of 100.

Our analogy with pages illustrates how z-index works. To determine how elements overlap, we assign them an integer. Larger integers appear on top and smaller integers appear below.

z-index Rules  

z-index has a few rules that determine how it affects stacking order, we should go over those ground rules of z-index to give us a firm ground for evaluating the following code samples.

  1. The values set to z-index should always be an integer (... -2, -1, 0, 1, 2, ...).
  2. z-index values (other than 0) can only be set on elements that have been positioned relative, absolute, or fixed
  3. The default z-index value for all elements is 0
  4. The higher the z-index, the higher an element will appear in the stacking order. i.e. if an element has a z-index value of 10 and another element has a z-index value of 5, then the first element will appear above the second element.

The z-index Property  

Setting z-index is as simple as assigning an element's z-index property an integer value. In the following code snippet, every p tag has had it's z-index set to 4 and the h1 tags have a z-index of 3. p has the greater z-index value, so if p and h1 tags overlap, the p tag will appear on top.

p {
  position: relative;
  z-index: 4;
}
h1 {
  position: relative;
  z-index: 3;
}

Now lets look at a few live code examples.

z-index Default Stacking Order  

To begin we'll look at the behavior or stacked elements before we change any z-index values. The default z-index value of elements is 0. This means that all elements have the same value. When the browser interprets how to render overlapping elements with the same z-index value, it looks at where the elements appear in the text document. The elements that appear first (nearer to the top of the document) will appear below the element that appear later (nearer the bottom of the document).

Basically, the browser starts at the top and works it's way down the page rendering each element in turn. This has the effect of placing later elements on top of earlier elements.

In the following example, the three colored squares have all been relatively positioned so that they overlap. The order of overlap is determined by the z-index value of the elements. Because, we haven't set an explicit z-index value, all the squares default to a z-index of 0. If overlapping elements have the same z-index value, then the browser will look at the order in which they appear in the HTML document.

In the following example, #blue appears first in the document, then #yellow, and finally #green. This order puts #blue on the bottom and #green on top.

Conclusion  

In this lesson, we learned about the z-index property. z-index allows us to control the stacking order of page elements that overlap and exert a finer control over how the browser renders page elements. z-index can be used to great effect when laying out pages, but it can also get complicated as you start trying to shuffle around multiple elements. Use it sparingly and you should be fine.
