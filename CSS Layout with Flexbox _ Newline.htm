Flexbox  

Flexbox is a fairly modern CSS technology that provides a series of properties to handle page layout. It is a direct replacement for using floats to construct layouts. It's a powerful tool for arranging elements but some of the properties aren't exactly intuitive. Still, it's a huge improvement over floats.

In this article, we will discuss the display: flex; property and several other properties related to controlling flex behaviors.

The first thing to recognize is that "Flexbox" is a collection of CSS properties to control layout behaviors. It isn't just one property. This collection of properties is designed to give you control over the way that elements expand vertically and horizontally. It also gives us direct control over whether elements line up horizontally or vertically.

Those properties fall into two broad categories:

  • Flex Container Properties - Properties applied to the parent of flex items to control the overall behavior of the entire group of flex items.

  • Flex Item Properties - Properties that are applied to elements that are children of a flex container. These elements can be arranged to create columns, rows, and grids for layout purposes.

Flexbox is a fairly complex subject, so this article will be presented in two large sections. In the first section, we will define all of the properties related to flex containers. Then, in the second section, we will discuss the properties associated with flex items. We will not discuss all of the available flex properties, instead focusing on the ones that have the most impact on layout. Once you've gotten a handle on these properties, you can look at the others at your leisure.

Before we begin discussing Flexbox properties, we need a baseline against which to compare their effects.

The following code sample has 9 divs nested within a section. Classes have been applied to the div elements to create a list of elements of varying widths. As we talk about the different CSS properties, we'll use this same HTML. So far, the code sample doesn't have any Flexbox applied to it, nor have the divs been arranged into a pleasing layout.

As we step through the properties, you will see all the values that can be applied. Each code sample will have one of the values applied so that you can see a new behavior. To see the behavior of the other values, change the CSS, and rerun the code sample.

Flex Containers Properties  

The flex container is an element that is the parent of child elements that we want to arrange with flexbox. The flex container has properties that allow it to control the overall behavior of those flex items. For instance, if we want the child elements to line up horizontally, we'd set the flex-direction property of the flex container to "row". If we want the child elements to be justified evenly on both sides, we set the justify-content property of the flex container to "space-between".

display flex  

Flex containers are created with display: flex. Any element that has a display value of "flex" is a flex container.

display: flex;

Just by applying display: flex; to the section element, we make it a flex container. By doing so, we make all of its children flex items. They immediately inherit all of the default properties for a flex element. They also abide by the default values for other flex container properties.

Flex Direction  

The flex-direction property tells flex items how to line up.

/* (The default value) tells flex items to line up horizontally from left to right. */
flex-direction: row;

/* tells flex items to line up vertically from top to bottom. */
flex-direction: column;

/* tells flex items to line up horizontally from right to left.
It's just the reverse of "row". */
flex-direction: row-reverse;

/* tells flex items to line up vertically from bottom to top.
The reverse of "column". */
flex-direction: column-reverse;

The default flex-direction value is row, which tells elements to line up horizontally from left to right. In the following example, flex-direction has been set to row-reverse, which forces flex items to line up horizontally from right to left. You can see this effect in the numbers.

Play with the code sample and change the flex-direction value to column and column-reverse to see what changes.

Flex Wrap  

The flex-wrap property determines in flex items will "wrap" to form multiple rows. The default value for this property is "nowrap" which forces elements to squish together to all fit onto one row.

/* (default): Tells all flex items will be on one line */
flex-wrap: nowrap;

/* Tells flex items will wraparound to new lines, arranged from top to bottom. */
flex-wrap: wrap;

/* Tells flex items will wrap around to new lines, arranged from bottom to top. */
flex-wrap: wrap-reverse;

In the following code sample, flex-wrap has been set to "wrap" which will allow the div elements to expand to their full width and wrap onto multiple lines. Notice that each div has a class applied to it that determines its width.

Width can be controlled on an element-by-element basis with Flex item properties like flex-basis. We'll get to that shortly.

Justify Content  

The justify-content property tells flex items how to align themselves with the left and right edges of the flex container and how to distribute any space remaining on each row.

/* (default) all flex items are aligned to the left of the flex container.
All remaining space is collected on the right side of the row */
justify-content: flex-start;

/* all flex items are aligned to the right of the flex container.
All remaining space is collected on the left side of the row */
justify-content: flex-end;

/* all flex items are aligned with the center of the flex container.
All remaining space is distributed equally on both sides of the row. */
justify-content: center;

/* All flex items in each row are distributed equally
across the full width of the flex container. Any remaining space is distributed
between the flex items  */
justify-content: space-between;

/* All flex items in each row are distributed equally
across the full width of the flex container. Any remaining space is distributed
between the flex items and the edges between the bounds of the
flex container and the flex items. The space between flex items is twice the size of
the space between the bounds of the flex container and the edge flex items */
justify-content: space-around;

In this example, justify-content has been set to "space-between", which causes the elements to justify themselves with both the right and left side of the flex container. The remaining space is distributed in the spaced between flex items. Try changing the justify-content property to the other values to see how the arrangement of elements is altered.

Align Content  

The align-content property determines how rows of flex items are aligned to the upper and lower bounds of the flex container, and how the remaining vertical space is distributed between rows.

/* (default); lines stretch to take up the remaining space */
align-content: stretch;

/* lines packed to the start of the container */
align-content: flex-start;

/* lines packed to the end of the container */
align-content: flex-end;

/* lines packed to the center of the container */
align-content: center;

/* lines evenly distributed; the first line is at the start of the container while the last one is at the end */
align-content: space-between;

/* lines evenly distributed with equal space around each line */
align-content: space-around;

In the following example, the height of the div elements (the flex items) has been shortened so that you can better see the behaviors of each align-content value. The height of the flex container has also been set to a fixed value, better to see the effects of each behavior. Try each value and see how it affects the layout of the flex item rows.

Align Items  

The align-items property tells flex items how to align themselves vertically and how to use any vertical space remaining in a column.

/* (default) Stretch vertically to fill
the full height of the flex container */
align-items: stretch;

/* Vertically align the tops of each flex item with the
upper bound of the flex container */
align-items: flex-start;

/* Vertically align the bottoms of each flex item with
the lower bound of the flex container */
align-items: flex-end;

/* Vertically align the center of each
flex item with the row that it is on */
align-items: center;

/* Vertically align the text baseline of
each flex item with the row that it is on */
align-items: baseline;

To better see the behaviors, we'll use a different example that incorporates text. This will give each div a different height and make it possible to see the effects of align-items. We'll also give the flex container a set height so that we can see the effects of vertical alignment.

Unfortunately, the following example uses a few flex item properties. They are necessary to see the effects of align-items. For now, ignore the flex styles applied to the div elements. You should focus on align-items and play with the values that can be set for that property.

The default value of align-items is "stretch" which forces all of the flex items to expand downward and fill any vertical height remaining in the flex container. Flex Item Properties  

Flex container properties give us broad control over the children of a flex container, but flex item properties allow us to control the behaviors of flex items on an element-by-element basis. This is really useful for creating complex layouts and for gaining fine control over our pages.

This section of the article will deal with those properties and how they can be used to manipulate the rendering of individual flex items.

Just to reiterate, up till now, we've been talking about properties that should be applied to the flex container. These next four properties should be applied to flex-items.

The order property allows us to arbitrarily shuffle the order in which page elements appear. The order property is assigned an integer (..., -2, -1, 0, 1, 2, ...). Flex Basis  

The flex-basis property determines the initial width of a flex item before it's been adjusted by flex-grow or flex-shrink. In many ways, it's similar to width but only applies within the context of a flex container. As such, it can be assigned the same types of values as width, e.g. percentages, pixel widths, ems, rems, etc.

The next few examples will use flex-basis instead of width.

Flex Grow  

Theflex-grow property tells flex items how they should expand to absorb any remaining width. Explaining this is harder than showing it, so we should look at a couple examples.

First, let us look at an example where the flex-grow value has been left unset. By default, flex-grow is set to 0.

In the next example, four flex items have been given a width of 10%, leaving 60% of the width of the flex container unfilled. Because flex-grow is 0, the flex items do not expand to absorb the remaining space, leaving a large unfilled space to the right of the flex items.

A flex-grow value of 0 means that the flex item cannot grow to fill empty space. You can see this in the first example.

Sibling flex items can be given different flex-grow values to create asymmetrical growth. You can think of flex-grow numbers greater than 1 in this way: "The number represents the amount of remaining space that an element will absorb relative to its siblings".

If one flex item has a flex-grow value of 2 and all the other flex items have a flex-value of 1, then the item with a flex-grow value of 2 will absorb twice as much of the available space as the other items.

In this next example, the flex-basis of each flex item is still 10%. All of the div elements have a flex-grow value of 1 except the first, which has a flex-grow value of 5. This means that the first element will absorb 5 times more of the available 60% width than the other elements. You can adjust the flex-grow value of #bigger to see the widths change accordingly.

Flex Shrink  

The flex-shrink property is similar to flex-grow, except that it controls how flex items will shrink if the row they are on overflows the width of the parent.

The values assigned to flex-shrink are similar to flex-grow as well. A flex-shrink value of 0 means that the flex item cannot shrink. If it overflows its parent, oh well. A flex-shrink value of 1 means that every element shrinks the same amount. flex-shrink values larger than 1 create asymmetrical shrinkage in the same way as different flex-grow values will create asymmetrical growth.

In this first example, the flex items overflow their parent because their flex-shrink has been set to 0, meaning that they cannot shrink.

Conclusion  

Learning to manipulate layouts with any efficiency will take time. Flexbox will give you a solid foundation for how to do layout and for how to control the location of screen elements. You'll need to practice this set of properties if you want to become very familiar with strategies for how best to use them.

Don't get stressed out if you don't understand every property by the end of this article. No one is expecting you to memorize all of these details. Use this lesson as a reference and as a guide for flexbox.

Building a Multi Column Layout with Flexbox  

You may have seen this design in the Layout with Floats lesson. We are walking you through the same mockup again, this time with flexbox, so that you can see how the two layout strategies (floats and flexbox) differ.

In this lesson, we will walk through the creation of a layout, using flexbox as the principle strategy for laying out the page. The purpose of this article is to demonstrate the thought process of constructing a layout and to demonstrate the authoring of CSS in a structured way.

First, we will review the mockup and the HTML. This will allow us to evaluate all of the content and look for clues about how best to style our page.

Once we have a firm understanding of the markup, we'll start adding the styles to lay out the page.

Reviewing the Mockup  

This is our design. Its a simple 3 column design with 6 content elements. Lets evaluate the major components of this layout to help us decide which items should be styled and flex containers and which items should be styled as flex items.

Creating the Content Container  

Never forget that everything in HTML is wrapped in a box. Sometimes, the box is obvious and sometimes the box is invisible. In this case, I see a fairly clear box around our entire content area. I look for places where multiple content elements line up. That's usually a good indicator of a common parent. Plus, if we add a common parent to all of the content, then we can move them all around together. I'll need an element to act as this ".wrapper".

Identifying Content Sections  

Now that I have a common parent, I look for logical breaks in the areas of content. In this case, we have three sections.

  1. SECTION 1 - The top row of content contains two pieces of content, one wide and one narrow. Its at the top of the doc, so will appear first in our HTML. We'll pretend that this section is the "main" content for the page. This section contains two elements that are side-by-side, so it'll be a flex-container.
  2. SECTION 2 - The middle row of content contains one paragraph of text. It has some special styling, so we'll need a way of selecting it independent of the other content areas. Let's pretend that this is a sage quote. This element only contains one child, so it doesn't need to be a flex container. We'll leave it display:block;
  3. SECTION 3 - The last row of content contains three pieces of content, all of which are styled in the same manner. We'll identify this as a list of cool things. This section contains three elements that are side-by-side, so it'll be a flex-container.

Identifying Columns  

When I look at this mockup, I can see three distinct columns. These columns give us a hint about how to style our content. For instance, the div containing "Page Title!" spans two columns. #quote spans 3 columns. The other four elements all span 1 column.

The width "220px" (or some multiple of it) will be something that we will need when are styling content.

Conclusion  

Now that we've walked through the process of evaluating and coding a flexbox layout, I encourage you to recreate this webpage on your own. You can follow along with this tutorial and copy the code into your own index.html and style.css pages.
